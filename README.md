# АРМ «Олимпиадное движение»

Автоматизированное рабочее место для ввода, хранения и обработки информации
об участии студентов в олимпиадном движении и конкурсах профессионального
мастерства.

Курсовая работа по дисциплине **«МДК 05.03 Тестирование информационных систем»**
(ГАПОУ СО «Екатеринбургский монтажный колледж»).

Тема: *«Создание и тестирование АРМ для обработки информации об участии студентов
в олимпиадном движении, конкурсах профессионального мастерства»*.

## Назначение

Система предназначена для:
- систематизации и информирования общественности о результатах студентов в
  олимпиадах и конкурсах профессионального мастерства;
- обеспечения механизма формирования отчётности по олимпиадам.

## Состав проекта

Проект состоит из двух самостоятельных частей:

| Часть | Папка | Технология | Назначение |
|-------|-------|-----------|------------|
| **1. REST-сервис** | [`olimp_api/`](olimp_api/) | FastAPI | REST API (архитектура MVC) |
| **2. Web-приложение** | [`olimp_django/`](olimp_django/) | Django | Веб-интерфейс (архитектура MVT) |

Обе части реализуют одну предметную область и могут использоваться независимо.

## Роли пользователей

| Роль | Возможности |
|------|-------------|
| **Незарегистрированный** | Просмотр списка олимпиад, фильтр по году, протоколы победителей (опубликованные), положения, задания прошлых лет |
| **Студент** | Запись на олимпиады, просмотр своих результатов, редактирование профиля |
| **Преподаватель** | Создание учётных записей студентов, прикрепление положений и заданий, ведение протоколов и результатов, подготовка/отзыв протоколов, бланки сертификатов/грамот, отчёт по протоколу |
| **Администратор** | Создание и редактирование учётных записей преподавателей, заполнение карточек олимпиад, публикация подготовленных протоколов, годовой отчёт |

**Жизненный цикл протокола:** `формируется` → `подготовлен` → `опубликован`.
После публикации протокол становится виден незарегистрированным пользователям.
Отозвать (вернуть в «формируется») можно только протокол со статусом «подготовлен».

## Аутентификация

Обе части используют вход **по логину и паролю** (без ручной вставки токенов):

- **FastAPI** — OAuth2 password flow. В Swagger UI (`/docs`) кнопка **Authorize**
  принимает логин и пароль, токен подставляется автоматически.
- **Django** — стандартные формы входа (`LoginView`) и самостоятельная
  регистрация студентов.

## Предметная область (модель данных)

- **Пользователь** — роль (администратор / преподаватель / студент), профиль.
- **Олимпиада** — название, описание, год, даты, уровень, ответственный
  преподаватель, положение, задания.
- **Запись (регистрация)** — связь «студент ↔ олимпиада».
- **Протокол** — результаты по олимпиаде, статус, ответственный преподаватель.
- **Результат** — баллы, место, тип (победитель / призёр / участник).
- **Сертификат/грамота** — бланки наград (к одной олимпиаде — любое количество).

## Часть 1. REST-сервис (FastAPI)

Расположение: [`olimp_api/`](olimp_api/). Подробности — в
[`olimp_api/README.md`](olimp_api/README.md).

Архитектура **MVC**:

| Слой | Каталог | Назначение |
|------|---------|------------|
| **Model** | `app/models/` (ORM) + `app/schemas/` (Pydantic) | Данные и их структура |
| **Controller** | `app/controllers/` | Бизнес-логика и работа с БД |
| **View** | `app/routers/` | HTTP-эндпоинты |

### Запуск

```bash
cd olimp_api
pip install -r requirements.txt
uvicorn app.main:app --reload
```

По умолчанию используется SQLite — настройка БД не требуется.
Документация API: http://127.0.0.1:8000/docs

### Тесты

```bash
cd olimp_api
pytest
```

## Часть 2. Web-приложение (Django)

Расположение: [`olimp_django/`](olimp_django/). Подробности — в
[`olimp_django/README.md`](olimp_django/README.md).

Архитектура **MVT**:

| Слой | Где |
|------|-----|
| **Model** | `accounts/models.py`, `olympiads/models.py` |
| **View** | `accounts/views.py`, `olympiads/views.py` |
| **Template** | `templates/` |

Контроль доступа по ролям — декораторы в `olympiads/decorators.py`.

### Запуск

```bash
cd olimp_django
pip install -r requirements.txt
python manage.py migrate
python manage.py seed_demo      # демо-данные и тестовые учётки (опционально)
python manage.py runserver
```

Приложение: http://127.0.0.1:8000/

### Тестовые учётные записи (после `seed_demo`)

| Роль | Логин | Пароль |
|------|-------|--------|
| Администратор | `admin` | `admin123` |
| Преподаватель | `teacher` | `teacher123` |
| Студент | `student` | `student123` |

### Отчёты

- Протокол по олимпиаде — `/report/protocol/<id>/` (CSV для Excel).
- Годовой отчёт участия — `/report/annual/<год>/` (CSV для Excel).
